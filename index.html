The Composite Score

From API Effectiveness as the Control Plane for AI (Khalsa, Nourian, Crawford, Gray).

AIEI = (wQ x Q) + (wL x L) + (wR x R) + (wC x C_norm)

Four orthogonal outcome dimensions, each bounded 0-1. The composite is a weighted sum, not a product - so one weak dimension drags the score but does not zero it out. The weights are configurable by organizational priority.

Design decisions: Multiplicative where failures cascade (R-Score). Comparative, not absolute (L-Score). Bounded 0-1 for composability. Deterministic and auditable - same inputs, same outputs, every time.

Dimension Formulas

Q: Quality
Q = [1 - (Defect_Rate_AI / Defect_Rate_NonAI)] x Test_Pass_Delta x (1 - Rework_Rate_AI)

Is AI-generated work production-grade? Multiplies three factors: defect improvement ratio, test pass rate delta, and rework avoidance. If any factor collapses (e.g. 40% rework rate), Q collapses. Below 0.5 means AI is creating technical debt faster than value.

L: Lead Time
L = 1 - (Cycle_Time_AI / Cycle_Time_NonAI)

Is AI actually faster end-to-end? Comparative, not absolute - measures cycle time delta between AI-assisted and non-AI work. L = 0.40 means 40% faster. L negative (not possible in calculator, but in real data) means AI is slower due to review overhead exceeding generation speed.

R: Reliability
R = (1 - CFR_AI) x SLO_Attainment x Contract_Catch_Rate x Uptime_AI x Adaptability

Is the system stable under AI acceleration? Product of five factors - if any one goes to zero, R goes to zero. This is how reliability actually works: one broken link breaks the chain. Postman can measure ~70% of R natively today (SLOs via Monitors, contract catch via CLI, uptime via API monitoring). R trending down while L trends up = velocity buying fragility.

C: Cost Efficiency
C_raw = (Labor_Value_Saved - Overhead_Value) / (Token_Cost + Infra_Cost + Tooling_Cost)
C_norm = min(C_raw / Target_ROI, 1.0)

Is AI worth what it costs, all-in? Includes the trust-but-verify tax (senior engineers reviewing AI output) that most organizations ignore. C_raw < 1.0 means AI costs more than it saves. Normalized against a target ROI to bound between 0 and 1. TCO typically exceeds expectations by 40-60%.

What AIEI Scores Mean

0.70 - 1.00: Effective
AI is measurably improving outcomes. Quality, speed, reliability, and cost are all contributing positively. Safe to expand AI adoption. Present results to the board with confidence. Focus shifts to optimization and scaling.
0.40 - 0.69: Mixed
AI is producing some value but specific dimensions need attention. Common pattern: speed improves while quality or reliability degrades. Look at the weakest dimension score - that is where the system is leaking. Do not expand AI to new teams until the weakest dimension is addressed. This is where most enterprises land today.
0.00 - 0.39: Technical Debt
AI is not yet effective as a system. The trust-but-verify tax likely exceeds the speed gains. Pause AI expansion. Invest in measurement infrastructure, governance automation, and contract testing before scaling further. This does not mean AI is bad - it means the surrounding system is not ready.

Dimension Interpretation Guide

Q (Quality): 0.70+ = production-grade, low rework. 0.40-0.69 = mixed, rework or test gaps. Below 0.40 = creating debt faster than value. Contract testing is the fix.
L (Lead Time): 0.50+ = 50%+ faster end-to-end. 0.25-0.49 = moderate, review overhead limiting gains. Below 0.25 = review tax eating generation gains.
R (Reliability): 0.70+ = system absorbing velocity safely. 0.40-0.69 = at least one weak factor dragging. Below 0.40 = fragile under AI velocity, pause expansion.
C (Cost Efficiency): 0.70+ = strong ROI including verify tax. 0.40-0.69 = returns exist but overhead consuming value. Below 0.40 = net cost, deployment model needs to change.

Measurement Maturity Model

Not a rollout plan. A model of rigor. The only thing that changes is what you can credibly claim.

Level 0: Anecdotes and usage counts. Sentiment only. This is where 77% of enterprises sit today.
Level 1: Attribution plus minimum scorecard. Lead time, defects, CFR, MTTR, token cost per valid artifact, trust-but-verify tax. First board-level conversation.
Level 2: Joins plus criticality weighting. Change-to-deploy-to-incident joins, API catalog weighting. AI impact on change safety for critical systems, not just averages.
Level 3: Guardrails plus unit economics. Overrides, hallucination incidents, policy exceptions, cost-to-serve proxies. This is where the MIT 95% failure rate reverses.
Level 4: Interface pipeline measurement. SDK release lag, adoption, parity, drift, integration time deltas. Safe consumption at scale for humans and agents.

The Measurement Contract (Non-Negotiables)

Attribution: AI-assisted vs non-AI tagging. Without attribution, deltas are stories, not evidence.
Baseline / Control: Deltas without baselines are meaningless. Need a comparison cohort.
Criticality Weighting: What matters most counts more. Not all APIs are equal.
Validity: Decided by real validation, not generation. Passing tests, not producing code.
Joins: Change to deploy to incident. API asset to Postman lifecycle to interface artifacts.
Anti-Vanity: AI activity is not AI effectiveness. Acceptance rates tell you nothing about outcomes.

If you cannot specify unit, cohort, validity, cost, and confounds, you cannot claim AI effectiveness.
